--- layout: post title: Generador de contenido estático Jekyll en Github Pages date: '2018-01-07T12:21:00.000+01:00' author: AITOR TOMAS PIN tags: modified_time: '2018-01-07T12:21:03.565+01:00' blogger_id: tag:blogger.com,1999:blog-611028137431305192.post-2055729653856092246 blogger_orig_url: http://compupiensaumh.blogspot.com/2018/01/generador-de-contenido-estatico-jekyll.html --- Después de leer el artículo de The Chronicle sobre construir una clase, projecto o cualquier otra página web para su uso en el aula con Jekyll y Github Pages ("Build a class, project, or other website using Jekyll + GitHub Pages") decidí realizar esta entrada en el Blog.

Yo siempre he sido un gran defensor de utilizar un generador de contenido estático como alternativa a los CMS (Content Management System) tradicionales del estilo de Wordpress. En particular, debido a mi conocimiento en Ruby, siempre he defendido el uso de Jekyll (a pesar de que hay otros muchos como Hugo en Go). Las razones son múltiples, pero por mentar algunas:
- No tener que utilizar una base de datos lo que simplifica la infraestructura sobremanera.
- El rendimiento y velocidad de renderizado de las páginas puesto que se trata de contenido estático.
- El uso de Markdown para crear nuestras páginas/posts.
- La flexibilidad para cambiar estilos, plantillas, etc.
- La posibilidad de hostear nuestra página/blog en Github de manera gratuita.

Este último punto es uno de los que me parecen más interesantes desde un punto de vista didáctico. Para cada usuario de Github, éste te permite crear un repositorio con el subdominio "<nombre de usuario>.github.io" en el cual se puede instalar Jekyll y Github se encargará de generar el contenido estático y renderizar tu página/blog generado en el subdominio mentado anteriormente.

Por tanto, sería posible crear una actividad de aula que consistiera en generar la página o blog del aula en particular involucrando a todos los alumnos. Como hemos comentado en posts anteriores, Git/Github es una combinación perfecta para trabajar de manera colaborativa sobre el mismo código fuente. Esto nos permitiría dividir a los alumnos del aula en distintos grupos con distintas responsabilidades (diseño, desarrollo, publicación de contenido, etc.) y realizar un proyecto conjunto para generar su propia página o blog.

Como ejemplo, considerando que simplemente he instalado el tema estándar de Jekyll, podéis encontrar los posts de este blog en Jekyll y Github Pages aquí: https://atomasp.github.io/
